# Backend: database

## Style

```python
class User(Base, IdMixin, TimestampMixin):
    __tablename__ = "users"
    email: Mapped[str] = mapped_column(String(320), unique=True, index=True, nullable=False)
```

Never legacy `Column(...)` without `Mapped[...]`. Mixins live in `db/mixins.py`:
`IdMixin` (UUIDv7), `TimestampMixin` (created_at/updated_at), `SoftDeleteMixin` (deleted_at),
`TenantMixin` (organization_id FK with cascade).

## Indexing rules (mandatory)

- **Every FK column gets an index** — Postgres does not index FKs automatically.
- **Every authenticated `WHERE` column gets an index** (email lookups, token_hash, prefix, etc.).
- **Tenant-scoped queries get a composite index** leading with `organization_id`:
  `ix_<table>_organization_id_<col>` (status, created_at, etc.).
- **"Latest N per parent" patterns** need a covering composite on `(parent_id, created_at)`.
  Don't rely on the FK index alone for ordered queries.
- **Partial indexes** for small "active" subsets:
  ```python
  Index("ix_x_active", "token_hash", postgresql_where=text("revoked_at IS NULL"))
  ```
- **Time-range scans** need a btree on the time column.
- **Don't pile on indexes.** Each index slows writes. Justify every one with an actual query.

## Query rules

- **Always include `organization_id`** in WHERE for tenant tables. Repo methods take it as a
  required parameter. There is no global helper that bypasses tenant scoping.
- **Bounded results.** Every list query has a LIMIT. Cursor pagination on indexed
  `(created_at, id)` — no OFFSET for user-facing endpoints.
- **No N+1.** Loading a parent + children uses `selectinload`/`joinedload` or batch-fetch by
  ID set with `WHERE id IN (...)`. Never loop and re-query.
- **Single round-trip per logical step.** Don't issue three sequential `await session.get(...)`
  when one join can produce the same result.
- **Transactions are explicit and short.** Repos `flush()`, services `commit()`. One session per
  request (FastAPI dep), commits once at end of unit of work, rolls back on exception.
- **No queries inside Python comprehensions** that hide await behavior.

## Migrations

- Every PR that changes models includes an Alembic migration with a hand-written `downgrade()`.
- Naming convention is configured in `db/base.py`; let Alembic emit constraint/index names
  from it. For hand-written indexes/constraints, follow `ix_<table>_<cols>` etc.
- Squashing pre-1.0 is allowed. After 1.0: never edit a released migration; always add a new one.
- Imports for new models in `domain/__init__.py` and `domain/<context>/__init__.py` so Alembic
  autogenerate sees them.

## Naming map (from `db/base.py`)

```python
NAMING_CONVENTION = {
    "ix": "ix_%(table_name)s_%(column_0_N_name)s",
    "uq": "uq_%(table_name)s_%(column_0_N_name)s",
    "ck": "ck_%(table_name)s_%(constraint_name)s",
    "fk": "fk_%(table_name)s_%(referred_table_name)s_%(column_0_name)s",
    "pk": "pk_%(table_name)s",
}
```

## Type map

`Base` declares a `type_annotation_map`:
- `UUID` → `Uuid(as_uuid=True)` (UUIDv7 generated by `uuid-utils`).
- `datetime` → `DateTime(timezone=True)`.
